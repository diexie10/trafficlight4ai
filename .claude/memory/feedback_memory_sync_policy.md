---
name: 记忆同步策略
description: 保存记忆时，无敏感信息的记忆要同步到项目 .claude/memory/；不同步的要说明原因
type: feedback
---

保存新记忆时，如果内容无敏感信息，必须同时同步到项目目录 `.claude/memory/` 并提交到 git。

不建议同步的记忆（含个人凭据、环境特定配置、插件特定行为），要向用户说明不同步的原因。

**Why:** 项目内的记忆可以被其他会话和协作者复用，避免每次重新沟通。

**How to apply:** 每次保存记忆时，判断是否含敏感内容。无敏感内容 → 同时写入主记忆和项目 `.claude/memory/`。有敏感内容 → 只写入主记忆，并告知用户不同步的原因。
